The `Modal` component provides a flexible and customizable modal dialog for React Native applications. It supports dynamic theming, customizable headers, and a trigger element to open and close the modal.

## Installation

* Open a Terminal in your project's folder and run:

## Props

The component accepts the following props:

### Required Props

- **`modalVisible`** (`boolean`):  
  Indicates whether the modal is currently visible.

- **`setModalVisible`** (`(value: boolean) => void`):  
  Function to toggle the visibility of the modal.

### Optional Props

- **`variant`** (`ThemeName`):  
  Specifies the theme variant for the modal. Defaults to the current app theme.

- **`lightColor`** (`string`):  
  Custom background color for light mode.

- **`darkColor`** (`string`):  
  Custom background color for dark mode.

- **`trigger`** (`React.ReactNode`):  
  The element that triggers the modal when clicked. If undefined, the modal must be controlled programmatically.

- **`headerTitle`** (`string`):  
  The title displayed in the modal's header. Defaults to `"Container"`.

- **`showHeader`** (`boolean`):  
  Whether to display the modal header. Defaults to `false`.

- **`containerStyle`** (`ViewStyle`):  
  Custom styles for the modal container.

- **`modalVisibleBackgroundOpacity`** (`number`):  
  Opacity of the background overlay when the modal is visible. Defaults to `0.7`.

- **`children`** (`React.ReactNode`):  
  The content displayed inside the modal.

- **`...otherProps`** (`ModalProps`):  
  Additional props passed to the underlying `RNModal` component.

## Behavior

1. **Theming**:
   - Adapts to the current theme using the `useTheme` hook.
   - Custom colors can be applied with `lightColor` and `darkColor`.

2. **Header**:
   - An optional header displays a title and a close button.
   - Controlled by the `showHeader` and `headerTitle` props.

3. **Trigger**:
   - The `trigger` prop renders an interactive element to toggle the modal.
   - If no trigger is provided, the modal visibility must be managed programmatically.

4. **Customization**:
   - `containerStyle` allows you to extend or override modal container styles.
   - `modalVisibleBackgroundOpacity` customizes the transparency of the modal backdrop.
