The `Switch` component is a flexible and themeable toggle switch designed for React Native applications. It supports various customization options, including colors, styles, and interaction callbacks.

### Step 1: Install [`Hyper-Native-Ui`](https://www.npmjs.com/package/hyper-native-ui)

* Open a Terminal in your project's folder and run: `hyper-native-ui`

## Props

The component accepts the following props:

### Required Props

- **`isChecked`** (`boolean`):
  Indicates whether the switch is in the "on" state.

- **`onValueChange`** (`(value: boolean) => void`):
  Callback function triggered when the switch is toggled.

### Optional Props

- **`themeScheme`** (`"light" | "dark"`):
  Overrides the current theme scheme. Defaults to the app's current theme scheme.

- **`variant`** (`"default" | ThemeName`):
  Specifies the color variant for the switch. Defaults to `"default"`.

- **`viewStyle`** (`ViewStyle`):
  Custom styles for the container wrapping the switch.

- **`switchStyle`** (`{ isChecked: string, unChecked: string, thumbColor: string, borderColor: string }`):
  Custom styles for the switch colors and thumb appearance.

- **`...props`** (`SwitchProps`):
  Additional props passed to the underlying `RNSwitch` component.

## Behavior

1. **Styling**:
   - The appearance of the switch is determined by the `variant` prop or the custom `switchStyle` object.
   - Colors adapt to the current theme by default but can be overridden using the `themeScheme` prop.

2. **Interaction**:
   - The switch can be toggled between "on" and "off" states, triggering the `onValueChange` callback.
   - Disabled states are styled with reduced opacity.

3. **Customization**:
   - Custom container styles can be applied using the `viewStyle` prop.
   - Custom switch styles can be defined with `switchStyle`.
