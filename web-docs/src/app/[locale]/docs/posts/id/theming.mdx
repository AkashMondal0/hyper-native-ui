## Theme Provider

The `ThemeProvider` component manages theming across your app, allowing you to toggle themes, change theme colors, and manage system-based theming. Below is the updated API and implementation guide.

### Usage

Wrap your app with the `ThemeProvider` component:

<CodeBlock className="my-24" 
    compact
    codeInstances={[{
        code: `
import React from 'react';
import { ThemeProvider } from 'hyper-native-ui';

const App = () => (
  <ThemeProvider initialScheme="system" themeName="Grey" enableThemedStatusBar>
    {/* Your application components */}
  </ThemeProvider>
);

export default App;
`,
        label: 'React Native',
        language: 'tsx'
    }]}
    copyButton
/>

### Props
- **`children`** (`ReactNode`): The child components wrapped by the provider. Default is `undefined`.
- **`initialScheme`** (`"light" | "dark" | "system"`): The initial color scheme to apply. Default is `"system"`.
- **`themeName`** (`string`): The initial theme name to use. Default is `"Grey"`.
- **`enableThemedStatusBar`** (`boolean`): Whether to enable dynamic status bar theming. Default is `false`.

### Context API

The `ThemeContext` provides the following methods and properties for managing themes:

#### Properties

- `currentTheme`: The current theme object based on the `themeName` and `colorScheme`.
- `themeName`: The name of the active theme.
- `themeScheme`: The active theme scheme (`"light"` or `"dark"`).
- `currentColorScheme`: Reflects whether the theme follows the system (`"system"`), light, or dark.
- `statusBarColor`: The current status bar color.

#### useTheme Hook Methods

- `toggleTheme(themeSchema?: "light" | "dark" | "system")`
  - Dynamically toggles the theme scheme.
  - Pass `"system"` to follow the system preference.

- `changeTheme(themeName: string)`
  - Updates the theme name while maintaining the current color scheme.

- `changeStatusBarColor(statusBarColor: string)`
  - Changes the status bar color.

- `initialTheme({ colorScheme, themeName })`
  - Sets the initial theme configuration during app startup.
