## Props

The `Statusbar` component supports the following props:

### Required Props

None of the props are strictly required as defaults are provided.

### Optional Props

- **`viewProps`** (`ViewProps`):  
  Props passed to the `View` that acts as a spacer for safe area insets.

- **`safeAreaTopPadding`** (`number`):  
  Additional padding to apply at the top for safe area insets. Defaults to `0`.

- **`variant`** (`"default" | "secondary" | "danger" | "warning" | "success" | "outline" | ThemeName`):  
  Determines the background color of the status bar. Defaults to `"default"`.

- **`themeScheme`** (`"light" | "dark"`):  
  Specifies whether to use light or dark content for the status bar. Defaults to the current app theme scheme.

- **`...otherProps`** (`StatusBarProps`):  
  Additional props passed to the `StatusBar` component.

## Behavior

1. **Theming**:  
   - Adapts the `StatusBar` color to the current theme or a specific theme variant.  
   - Uses the `useTheme` hook to fetch theme colors dynamically.

2. **Safe Area Insets**:  
   - Integrates with `react-native-safe-area-context` to account for the device's safe area.  
   - A `View` spacer ensures the status bar does not overlap with content.

3. **Variants**:  
   - Includes predefined variants (`default`, `secondary`, `danger`, `warning`, `success`, etc.) and support for custom theme colors.

4. **Customization**:
    - Accepts additional props for the `StatusBar` component, such as `backgroundColor`, `barStyle`, and `translucent`.

5. **Theme Scheme**:
    - Supports light and dark theme schemes for the status bar content.

6. **Safe Area Padding**:
    - Allows adding extra padding to the top of the status bar to accommodate custom designs.
