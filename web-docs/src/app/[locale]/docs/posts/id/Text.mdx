The `Text` component is a versatile and theme-aware typography element designed for React Native applications. It supports various text styles, colors, alignment, and other customization options.

## Installation

* Open a Terminal in your project's folder and run:

## Props

The component accepts the following props:

### Required Props

- **`children`** (`React.ReactNode`):
  The content of the text.

### Optional Props

- **`variant`** (`"H1" | "H2" | "H3" | "H4" | "H5" | "H6" | "subtitle1" | "subtitle2" | "body1" | "body2" | "button" | "caption" | "overline"`):
  Specifies the text style variant. Defaults to `"body1"`.

- **`variantColor`** (`"default" | ThemeName`):
  Specifies the color variant for the text. Defaults to `"default"`.

- **`themeScheme`** (`"light" | "dark"`):
  Overrides the current theme scheme. Defaults to the app's current theme scheme.

- **`bold`** (`TextStyle['fontWeight']`):
  Specifies the font weight.

- **`size`** (`TextStyle['fontSize']`):
  Specifies the font size in pixels.

- **`center`** (`boolean`):
  If `true`, centers the text horizontally. Defaults to `false`.

- **`disable`** (`boolean`):
  If `true`, dims the text to indicate a disabled state. Defaults to `false`.

- **`style`** (`TextStyle`):
  Additional custom styles for the text.

- **`...otherProps`** (`TextProps`):
  Additional props passed to the underlying `Text` component.

## Behavior

1. **Typography Variants**:
   - The `variant` prop determines the text's font size, weight, and spacing based on Material Design guidelines.

2. **Styling**:
   - Text color adapts to the current theme but can be overridden using the `variantColor` and `themeScheme` props.

3. **Accessibility**:
   - The `disable` prop reduces text opacity to indicate non-interactive or disabled states.
